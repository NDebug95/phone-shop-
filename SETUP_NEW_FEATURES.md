# สิ่งที่เพิ่มเข้ามาใหม่ — วิธีติดตั้งให้ครบ

## 1. อัปโหลดโค้ด
อัปโหลดทุกไฟล์ในโฟลเดอร์นี้ทับของเดิมใน GitHub repo (Add file → Upload files)
แล้ว commit → Vercel deploy ให้อัตโนมัติ

## 2. ฐานข้อมูล — ทำให้แล้ว ไม่ต้องทำอะไรเพิ่ม ✅
ผมสร้างตาราง `profiles`, `orders`, `reviews`, `site_settings` และ storage bucket
`payment-slips` ในโปรเจกต์ Supabase ของคุณเรียบร้อยแล้ว พร้อม Row Level Security
(ลูกค้าเห็นแค่ออเดอร์ตัวเอง แอดมินเห็นทั้งหมด)

**แก้ช่องโหว่ที่เจอ:** นโยบายเดิมของตาราง `products` เปิดให้ "ผู้ล็อกอินทุกคน"
แก้ไข/ลบสินค้าได้ (ไม่ได้เช็คสิทธิ์แอดมินจริง) ตอนนี้เพิ่มระบบสมัครสมาชิกลูกค้าแล้ว
จึงต้องแก้เป็นเฉพาะแอดมินเท่านั้น — แก้เรียบร้อยแล้ว

## 3. ตั้งค่า QR รับเงิน
เข้า `/admin/settings` (ต้องล็อกอินแอดมินก่อน) → อัปโหลดรูป QR PromptPay/QR ธนาคาร
ลูกค้าจะเห็นรูปนี้ทุกครั้งตอนเช็คเอาท์

## 4. ตั้งค่าแจ้งเตือน LINE (ไม่บังคับ — ถ้าไม่ตั้งค่า เว็บยังใช้งานได้ปกติ แค่ไม่มีข้อความ LINE)

⚠️ **สำคัญ:** LINE ยกเลิกบริการ "LINE Notify" ไปแล้วตั้งแต่ 31 มี.ค. 2025
ต้องใช้ "LINE Messaging API" แทน วิธีตั้งค่า:

1. เข้า https://developers.line.biz/console/ → สร้าง Provider ใหม่ (ถ้ายังไม่มี)
2. สร้าง Channel ประเภท **Messaging API**
3. ในหน้า Channel → แท็บ "Messaging API" → เลื่อนลงไปหา **Channel access token**
   กด "Issue" เพื่อสร้าง token แล้ว copy เก็บไว้
4. เปิดแอป LINE บนมือถือคุณ → ในหน้า Channel เดียวกันจะมี QR code ของ
   LINE Official Account นี้ → สแกนเพื่อเพิ่มเป็นเพื่อน (ต้องเพิ่มเพื่อนก่อนถึงจะรับข้อความได้)
5. ไปที่ Vercel → Settings → Environment Variables → เพิ่มตัวแปรใหม่:
   - Key: `LINE_CHANNEL_ACCESS_TOKEN`
   - Value: token ที่ copy มาจากข้อ 3
   - **ไม่ต้องขึ้นต้นด้วย NEXT_PUBLIC_** (เพราะเป็นความลับ ต้องใช้แค่ฝั่งเซิร์ฟเวอร์)
6. Redeploy โปรเจกต์

ทุกครั้งที่มีลูกค้าสั่งซื้อ ระบบจะส่งข้อความ broadcast เข้า LINE OA ของคุณอัตโนมัติ
(นอกจากนี้ยังมีแจ้งเตือนในหน้า `/admin/dashboard` และ `/admin/orders` เสมอ
แม้ยังไม่ได้ตั้งค่า LINE)

## 5. หน้าเว็บที่เพิ่มเข้ามา
- `/products` — สินค้าทั้งหมด (ค้นหา + กรองยี่ห้อ)
- `/brand/[ยี่ห้อ]` — สินค้าตามยี่ห้อ
- `/reviews` — รีวิวลูกค้า (เขียนรีวิวได้เมื่อล็อกอิน)
- `/login`, `/register` — ระบบสมาชิกลูกค้า (แยกจากระบบแอดมิน)
- `/account` — ประวัติคำสั่งซื้อของลูกค้า
- `/checkout/[id]` — หน้าชำระเงิน (QR + แนบสลิป)
- `/admin/orders` — แอดมินตรวจสอบ/ยืนยันออเดอร์
- `/admin/settings` — อัปโหลด QR รับเงิน
